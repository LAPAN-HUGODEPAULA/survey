## Development Commands

### Version Management
- Sync version across managed files: `npm run version:sync`
- Check version consistency: `npm run version:check`
- Prepare for release: `npm run release:prepare`

### Flutter Development
- Survey frontend (Screener): `npm run dev:survey-frontend` (port 8080)
- Patient app: `npm run dev:survey-patient` (port 8081)
- Clinical narrative: `npm run dev:clinical-narrative` (port 8082)
- Survey builder: `npm run dev:survey-builder` (port 8083)

### Backend & Services
- Start all core services: `./tools/scripts/compose_local.sh up -d mongodb survey-backend survey-frontend survey-patient clinical-narrative survey-builder survey-worker`
- Start Clinical Writer (optional): `./tools/scripts/compose_local.sh up -d clinical-writer-api`
- Backend health check: `python -m compileall services/survey-backend/app`
- Regenerate SDKs: `tools/scripts/generate_clients.sh`

### Runtime Configuration
- Configuration source-of-truth: `config/runtime/config.private.json`
- Generate runtime artifacts: `python3 tools/scripts/render_runtime_config.py`
- Copy from example: `cp config/runtime/config.private.example.json config/runtime/config.private.json`

### Testing
- Run Flutter tests from app directories (e.g., `cd apps/survey-patient && flutter test`)
- Backend tests: Python pytest in `services/survey-backend`

## Architecture Overview

### Monorepo Structure
- **services/**: Python FastAPI backend and workers
  - `survey-backend`: Main API with MongoDB repositories, JWT auth, 10+ route modules
  - `survey-worker`: Background processor for response enrichment
  - `clinical-writer-api`: LLM-based clinical narrative generation
- **apps/**: Flutter web applications using shared design system
  - `survey-frontend`: Screener interface for administering surveys
  - `survey-patient`: Patient-facing survey completion
  - `clinical-narrative`: Clinical report viewer
  - `survey-builder`: Admin interface for creating surveys and prompts
- **packages/**: Shared packages
  - `contracts`: OpenAPI spec with generated Python/Dart SDKs
  - `design_system_flutter`: Reusable UI components and themes
  - `shared_python**: Common Python utilities

### Key Architecture Patterns
- **Repository Pattern**: MongoDB access through abstracted repositories in `services/survey-backend/app/persistence/repositories/`
- **Event-Driven**: Async background processing via survey-worker
- **Prompt-Based AI**: Clinical Writer uses questionnaire-specific prompts from MongoDB
- **Role-Based Access**: Multiple screener/admin roles with JWT authentication
- **Contract-First**: OpenAPI definitions drive client generation

### Core Data Models
- Surveys with dynamic questionnaires and personas
- Patient responses with encrypted PII
- Clinical reports generated by LLM
- Prompt catalog for questionnaire-specific AI logic
- Role-based access control

### Important Design Contracts
- Full-screen Flutter pages must use `DsScaffold` from `design_system_flutter`
- Shared theme uses `Colors.orange` as seed color
- API responses follow strict schemas from OpenAPI contracts
- Runtime config injected via environment variables

### Backend API Organization
- Modular route structure (separate files for each domain)
- Global exception handling with custom error mapper
- MongoDB connection pooling with health checks
- System screener auto-creation on startup
- CORS middleware for web frontend access
